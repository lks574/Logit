# Logit UX 고도화 — 벤치마킹 & 개선안 (트랙 A)

> 작성: 2026-07-22. 대상 워크트리 `lks574/enhance-ux`.
> 범위: (1) 유저 사용성 — 외부 앱 벤치마킹으로 **입력 마찰 감소**, (2) 우리앱만의 특장점 — **단일 타임라인 + 약속→기록 루프** 차별화.
>
> **판별 기준(ROADMAP §전략 준수): "타이핑을 줄이면 O · 화면·기능을 늘리면 X."**
> non-goals(휴식타이머·운동DB·루틴추천·자체GPS·예매파싱)에 걸리는 아이디어는 각 표 하단에 제외 사유를 명시한다.

---

## 0. 요약 — 무엇을 왜 골랐나

벤치마킹 결과, Logit이 이미 잘 갖춘 것(프리필 배너·최근 장소 칩·빠른 기록 칩·진행형 폼·회고 하이라이트)과, **경쟁 앱 대비 아직 비어 있는 마찰 포인트**가 갈렸다. 아래에서 발견한 개선안 중, 실제로 구현한 것은 **저비용·고효과이면서 "타이핑/탭 감소" 기준에 정확히 부합**하는 두 가지다.

1. **온보딩 선택 활동을 홈 빠른 기록에 반영** — 온보딩이 "고른 활동을 홈에 먼저 올려둘게요"라 약속하지만, 현재 홈은 `preferredActivities`를 **전혀 읽지 않는다**(기록 빈도 기반 `topActivities`만 사용). 신규 유저는 기록이 0이라 빠른 기록 칩이 **아예 안 보이고**, 첫 기록까지 `FAB → 활동선택 → 스크롤 → 탭`을 거친다. → 약속 이행 + 신규 유저 첫 기록 마찰 제거.
2. **지난 약속(overdue) → 기록 프롬프트를 홈에 노출** — 약속 날짜가 지났는데 기록으로 전환 안 한 약속은 홈 "다가오는 약속"에서 **조용히 사라진다**(`upcomingPlans`가 미래만 필터). "약속→기록 루프"의 핵심 연결이 끊겨 있다. → 루프를 홈 앵커에서 닫고, 한 탭 전환(폼 프리필)으로 마찰 제거.

두 항목 모두 **기존 패턴·스토어 필드·셀렉터 재사용, 신규 dep 0, 신규 화면 0.** 상세는 §3.

## 1. 유저 사용성 (입력 마찰 · 온보딩 · 빠른 기록 · 검색/필터 · 기본값 · 빈 상태)

### (a) 외부 앱이 잘하는 것
- **Hevy / 번핏**: "지난 워크아웃 반복(Repeat)" 1탭 — 직전 세트 구성을 그대로 불러와 수치만 수정. 기록당 실입력 시간을 10초대로.
- **Strava / NRC**: 운동 자동 수집(GPS·센서) → 사용자는 확인만. 손 입력 자체를 제거.
- **Letterboxd / 왓챠피디아**: 작품 검색 → 메타데이터 자동 채움. 별점 슬라이더/탭 한 번. 재관람은 기존 항목에 "다시 봄" 추가.
- **Day One**: 열자마자 "오늘" 컨텍스트(날짜·위치·날씨) 자동 프리필, 커서가 본문에 바로. 진입=쓰기.
- **Notion / 모먼트**: 템플릿·최근 항목을 상단에 노출해 반복 입력을 "고르기"로 치환.

### (b) Logit 현재 상태
- 프리필 배너(`PrefillBanner`), 최근 장소 칩(`recentValues`), 홈 빠른 기록 칩(빈도 top4), 진행형 공개 폼, 종목별 자동 계산(속도·칼로리), 날짜=오늘·시간=지금 기본값 — **핵심 마찰 감소 장치는 이미 상당수 구축**.
- 온보딩에서 활동을 고르지만(`preferredActivities`) 홈·활동선택 어디서도 **활용 안 됨**(빈 신규 유저 = 빠른 기록 칩 0).
- 검색은 활동선택에만 존재. 기록 본문(작품·장소·메모) 검색은 없음.
- 자유 입력 단위 미강제("5.2km"/"27:12") — 프로파일별 파서로 흡수 중이나 형식 오차 여지.

### (c) 개선 제안

| 제목 | 기대효과 | 예상규모 | 우선순위 |
|---|---|---|---|
| **온보딩 선택 활동 → 홈 빠른 기록 반영** ✅구현 | 신규 유저 첫 기록까지 탭 수 ↓, 온보딩 약속 이행 | 반나절 | **High** |
| 활동선택 "최근 사용" 행에도 preferredActivities 병합 | 기록 0인 신규 유저도 활동선택 즉시 개인화 | 반나절 | Med |
| 폼 진입 시 커서 자동 포커스(첫 필수 필드) + 키보드 즉시 | "진입=쓰기"(Day One). 탭 1회 절감 | 반나절 | Med |
| 최근값 칩 확대: 동행·작품·상대까지(현재 장소·공연장만) | 재입력 타이핑 추가 절감. `recentValues` 재사용 | 반나절 | Med |
| 기록 검색(작품/장소/메모 부분일치) — 통계·캘린더 진입점 | 과거 기록 재열람 마찰 ↓ (Letterboxd 라이브러리) | 1–2일 | Med |
| 별점을 스와이프/드래그로(현 탭 5회) | 평점 입력 손동작 1회로 | 반나절 | Low |
| 홈 인사 문구 시간대 적응(아침/저녁) | 심리적 진입 유도(정량 효과는 낮음) | 반나절 | Low |

**non-goals 제외**
- ~~GPS 자동 트래킹(Strava/NRC)~~ → 자체 GPS 금지. **HealthKit 가져오기(P3)**가 90% 대체 → 로드맵 P3로 이관.
- ~~작품/공연 메타데이터 API 자동 채움(Letterboxd)~~ → 외부 예매/작품 DB 파싱은 "화면 변경마다 깨짐" non-goal. 대신 **내 과거 기록 기반 최근값 칩**(위 표)으로 자체 데이터만 재사용.
- ~~단위 강제 입력 마스크~~ → 자유 입력 철학과 충돌. 현 파서 흡수 유지(스코프 밖).

### (d) 빈 상태 점검
- 홈/통계/약속 빈 상태 CTA는 이미 양호. 단 **신규 유저 홈의 빠른 기록 칩이 0개**라 "무엇을 기록하지?" 선택 마찰이 남음 → §3 QW1이 해소.

## 2. 우리앱만의 특장점 (단일 타임라인 + 약속→기록 루프)

### (a) 외부 앱이 잘하는 것 (그러나 버티컬이라 못 하는 것)
- 버티컬 앱들은 **한 종류**만 잘한다(Hevy=근력, Strava=러닝, Letterboxd=영화). 이종 활동을 한 타임라인에 못 섞는다.
- "약속(예정) → 실제 기록" 루프를 갖춘 로거는 드묾. 캘린더 앱은 예정만, 로거 앱은 기록만.

### (b) Logit 현재 상태 — 차별화 자산과 누수
- **자산**: 5템플릿·평평한 활동 카탈로그로 운동·공연·맛집을 한 홈/캘린더/통계에 병치. 약속→기록 전환(체크 → 폼 프리필 → 저장 시 약속 완료)도 홈·약속목록·캘린더 3곳에 배선됨.
- **누수 ①(핵심)**: 약속 날짜가 지나면 홈에서 사라진다. `upcomingPlans`는 미래만. 지난 미완료 약속은 **약속목록 탭의 '지난' 세그먼트에 숨어야** 보임 → 홈 앵커에서 루프가 끊김. → §3 QW2가 닫음.
- **누수 ②**: 홈 "이번 주" 구성 바가 이종 활동을 색으로 보여주는 건 좋으나, **"약속 대비 이행률"** 같은 루프 특화 지표는 없음.

### (c) 개선 제안

| 제목 | 기대효과 | 예상규모 | 우선순위 |
|---|---|---|---|
| **지난 약속 → 기록 프롬프트를 홈에 노출** ✅구현 | 약속→기록 루프를 홈에서 닫음. 이탈한 약속 회수 | 반나절 | **High** |
| 약속 이행률 위젯(이번 달 약속 N건 중 M건 기록) | 루프를 "습관 지표"로 승격, 우리만의 통계 | 1–2일 | Med |
| 타임라인 통합 필터(운동/공연/여가 토글)를 홈 최근 기록에 | 단일 타임라인 강점을 직접 체감 | 1–2일 | Med |
| "N일 전 오늘" 회고(작년/지난달 같은 날 기록) | 이종 타임라인만의 회고 서사(Day One "On this day") | 1–2일 | Med |
| 약속 카드에서 바로 "완료(기록 없이)"/"연기" 스와이프 | 지난 약속 정리 마찰 ↓ | 반나절 | Low |

**non-goals 제외**
- ~~종목별 심화 통계(상대 전적·루틴)~~ → ROADMAP 버티컬 심화 게이트(P4 데이터 이후)에 종속. 지금 착수 금지.

## 3. 구현한 Quick-win (2건)

### QW1 — 온보딩 선택 활동을 홈 빠른 기록 칩에 반영
- **문제**: `OnboardingScreen`은 "고른 활동을 홈에 먼저 올려둘게요"라 안내하고 `completeOnboarding(selected)`로 `preferredActivities`를 저장하지만, `HomeScreen`은 이 값을 읽지 않는다. 홈 빠른 기록 칩은 **기록 빈도(top4)** 로만 구성돼, 기록이 0인 신규 유저에게는 칩 행이 **통째로 숨겨진다**(`topActivities.length > 0` 가드).
- **해결**: 홈의 빠른 기록 목록을 `빈도순 → preferredActivities(중복 제거) 병합`(최대 6개)으로 확장. 기록이 없어도 온보딩 선택이 즉시 칩으로 노출돼 **1탭으로 폼 직행**. 기존 기록이 쌓이면 자연히 빈도순이 앞선다.
- **재사용**: 기존 칩 렌더·`colorsFor`·`activities`/`customActivities` 템플릿 해석. 스토어 `preferredActivities`(이미 존재). 신규 dep/화면 0.
- **파일**: `src/screens/home/HomeScreen.tsx` (칩 소스 계산부만 수정).

### QW2 — 지난 약속(overdue) → 기록 프롬프트를 홈에 노출
- **문제**: `upcomingPlans`가 `dayDiff(today, dateISO) >= 0`(미래·오늘)만 반환 → 날짜가 지난 미완료 약속은 홈에서 사라진다. 약속→기록 루프가 홈 앵커에서 끊긴다(약속목록 '지난' 탭까지 들어가야 보임).
- **해결**: 순수 셀렉터 `overduePlans(plans, today)` 추가(미완료·과거, 최근 지난 순). 홈 "다가오는 약속" 위에 **"이 약속, 기록했나요?"** 프롬프트 섹션(최대 3건) 노출. 각 행 탭 → 해당 약속으로 `RecordForm` 프리필(저장 시 `completePlan`으로 루프 종결). "전체 보기" → 약속목록.
- **재사용**: 홈 기존 `PlanRow`/전환 네비 패턴, `dday`·`slashDayWeekday`·`displayTimeLabel`, warning 색 토큰. 비파괴적(삭제·자동완료 없음 — 사용자가 전환하거나 약속목록에서 관리). 신규 dep/화면 0.
- **파일**: `src/store/selectors.ts`(셀렉터 추가), `src/screens/home/HomeScreen.tsx`(프롬프트 섹션).

## 4. 구현·검증

### 변경 파일
- `src/store/selectors.ts` — `overduePlans(plans, today)` 순수 셀렉터 추가(미완료·과거, 최근 지난 순).
- `src/screens/home/HomeScreen.tsx` — (QW1) 빠른 기록 칩 소스에 `preferredActivities` 병합(빈도순 뒤에 중복 제거, 최대 6). (QW2) "이 약속, 기록했나요?" 지난 약속 프롬프트 섹션 + `OverdueRow` 컴포넌트.

### 검증 로그
- **`tsc --noEmit`: 클린**(exit 0). `strict: true` 환경에서 통과, 신규 타입 오류 없음.
- **`expo export --platform web`: 성공**(exit 0, `Exported: dist`). 전체 Metro 웹 번들이 수정 코드 포함해 컴파일 에러·미해결 import 없이 빌드됨 — 웹 프리뷰 부팅 가능 상태 확인.
- **의존성**: 신규 dep 0. 신규 화면 0. 기존 컴포넌트/셀렉터/스토어 필드만 재사용.
- **비파괴성**: QW2는 삭제·자동완료 없음(전환 또는 약속목록 관리로만 소진). QW1은 preferred가 비면 기존 빈도 동작과 동일(회귀 없음).

### 검증 데이터 상 유의점(수동 클릭 확인용 참고)
- 실행 시 `today = nowDateISO()`(실제 오늘)라, seed 약속 4건(06-30~07-05)이 모두 과거 → **개발/웹에서 QW2 프롬프트에 즉시 노출**되어 육안 확인 가능(전환 시 폼 프리필·저장 후 소진).
- QW1의 `preferredActivities`는 seed에서 `[]`이므로, 온보딩을 완료해 활동을 골라야 신규 유저 시나리오(기록 0 + 칩 노출)를 육안 확인할 수 있음. 코드 경로는 tsc·번들로 검증됨.

### 한계(정직한 기록)
- 브라우저 자동 클릭 드라이버가 이 실행 환경에 없어, **인터랙션 육안 확인은 미실시**. 정적 검증(tsc·웹 번들)과 로직 리뷰까지 완료. 코디네이터/후속 세션에서 `npm run web`로 홈 화면의 (1) 빠른 기록 칩, (2) 지난 약속 프롬프트 탭→폼 프리필을 최종 육안 확인 권장.
