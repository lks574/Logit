# 노트 템플릿

## 학습 노트 (01~08 폴더)

```markdown
# 제목

> [!abstract] 한 줄 요약
> 이 노트가 답하는 질문.

## 🔁 iOS/AOS 대응 개념
| RN | iOS | Android |
|---|---|---|

## 🧭 왜 이렇게 설계됐나
(역사적/기술적 배경. "왜"를 모르면 외운 것이고, 알면 이해한 것이다)

## ⚙️ 동작 원리
(본문. 꼼꼼하게. 그림이 필요하면 mermaid)

## 💻 코드 예시
(실행 가능한 최소 예시. 기준 버전 명시)

## ⚠️ 함정 (Pitfalls)
(네이티브 개발자가 특히 걸리는 지점)

## 🔗 관련 노트
[[용어]] · [[용어]] · 다음 노트 링크
```

## 용어집 노트 (00-용어집)

```markdown
# 용어명

> [!abstract] 한 줄 정의
> …

> [!info] iOS/AOS로 치면
> …

## 📖 설명
(3~10문단. 처음 보는 사람이 이 노트만 읽고 대화에 낄 수 있는 수준)

## 🔗 관련
[[용어]] · [[용어]]
```

## 작성 규칙

- 표준 섹션만 이모지(🔁🧭⚙️💻⚠️🔗📖)를 붙인다. 노트별 커스텀 섹션은 이모지 없이 — 아이콘이 "표준 섹션" 표시로 기능하게.
- 용어 노트 파일명 = 용어 그대로 (예: `Hermes.md`). 본문 어디서든 `[[Hermes]]`로 링크 가능.
- 코드 예시는 TypeScript 기본. 네이티브 코드는 Swift / Kotlin.
- 버전 민감 내용은 기준 버전 명시 (예: "RN 0.76+", "Expo SDK 57").
- 확신 없는 내용은 쓰지 않는다. 공식 문서 링크로 대체.
- 함정(Pitfall)을 발견하면 해당 노트 + [네이티브 개발자의 함정](99-실습-프로젝트/네이티브-개발자의-함정.md)에 모두 기록.
